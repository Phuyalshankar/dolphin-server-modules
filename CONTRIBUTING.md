# Contributing to Dolphin Framework 🐬

First off, thank you for considering contributing to Dolphin! It's people like you that make Dolphin such a great tool.

## 🚀 How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issues to report bugs.
- Describe the bug in detail and provide a minimal reproduction if possible.

### Suggesting Enhancements
- Open a GitHub Issue with the tag `enhancement`.
- Explain why this feature would be useful to most Dolphin users.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes (`npm test`).
4. Format your code according to the existing style.
5. Create a Pull Request with a clear description of your changes.

## 🛠️ Development Setup

1. Clone your fork.
2. Install dependencies: `npm install`.
3. Build the project: `npm run build`.
4. Run tests: `npm test`.

## 📜 Code of Conduct
By participating in this project, you agree to abide by our Code of Conduct. Keep it professional, respectful, and inclusive.
